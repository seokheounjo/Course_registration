🎯 수강신청 핵심 서비스

@Service @Transactional public class EnrollmentService { public Enrollment enrollSubject( Long studentId, Long timetableId) { // 1. 학생/시간표 조회 Student student = studentRepository .findById(studentId) .orElseThrow(); Timetable timetable = timetableRepository .findById(timetableId) .orElseThrow(); // 2. 중복 수강 검증 boolean exists = enrollmentRepository .existsByStudentAndSubject( student, timetable.getSubject()); if (exists) { throw new RuntimeException( "이미 수강 중인 과목입니다"); } // 3. 시간 충돌 검증 List<Enrollment> conflicts = enrollmentRepository.findTimeConflicts( student, timetable.getDay(), timetable.getStartTime(), timetable.getEndTime()); if (!conflicts.isEmpty()) { throw new RuntimeException( "시간이 겹치는 과목이 있습니다"); } // 4. 정원 확인 long count = enrollmentRepository .countByTimetable(timetable); if (count >= timetable.getCapacity()) { throw new RuntimeException( "정원이 초과되었습니다"); } // 5. 수강신청 저장 return enrollmentRepository.save( new Enrollment(student, timetable.getSubject(), timetable)); } }
💡 핵심 설계:
• @Transactional로 원자성 보장
• 4단계 검증으로 데이터 무결성
• 명확한 예외 처리

🌐 REST API 컨트롤러

@Controller @RequestMapping("/student") public class StudentController { @PostMapping("/enroll") public String processEnrollment( @RequestParam Long timetableId, HttpSession session, RedirectAttributes redirectAttributes) { try { // 1. 세션에서 로그인 사용자 확인 String username = (String) session .getAttribute("username"); if (username == null) { redirectAttributes.addFlashAttribute( "errorMessage", "로그인이 필요합니다"); return "redirect:/login"; } // 2. 학생 정보 조회 Student student = studentService .getStudentByStudentId(username); if (student == null) { redirectAttributes.addFlashAttribute( "errorMessage", "학생 정보를 찾을 수 없습니다"); return "redirect:/student/enroll"; } // 3. 수강신청 처리 Enrollment enrollment = enrollmentService .enrollSubject(student.getId(), timetableId); redirectAttributes.addFlashAttribute( "successMessage", "수강신청이 완료되었습니다: " + enrollment.getSubject().getName()); } catch (RuntimeException e) { redirectAttributes.addFlashAttribute( "errorMessage", e.getMessage()); } return "redirect:/student/enroll"; } }
🔒 웹 계층 보안:
• 세션 기반 인증 및 권한 검증
• try-catch로 예외 처리
• PRG 패턴으로 중복 요청 방지

📊 복잡한 JPA 쿼리

@Repository public interface EnrollmentRepository extends JpaRepository<Enrollment, Long> { // 시간 충돌 검사 JPQL @Query( "SELECT e FROM Enrollment e " + "WHERE e.student = :student " + " AND e.timetable.day = :day " + " AND (" + " (e.timetable.startTime <= :startTime " + " AND e.timetable.endTime > :startTime) " + " OR " + " (e.timetable.startTime < :endTime " + " AND e.timetable.endTime >= :endTime) " + " OR " + " (e.timetable.startTime >= :startTime " + " AND e.timetable.endTime <= :endTime)" + " )" ) List<Enrollment> findTimeConflicts( @Param("student") Student student, @Param("day") String day, @Param("startTime") String startTime, @Param("endTime") String endTime); // 중복 수강 확인 boolean existsByStudentAndSubject( Student student, Subject subject); // 과목별 수강 인원 조회 long countByTimetable(Timetable timetable); // 학생별 수강 과목 (시간순 정렬) @Query("SELECT e FROM Enrollment e " + "WHERE e.student = :student " + "ORDER BY e.timetable.day, " + " e.timetable.startTime") List<Enrollment> findByStudentOrderBySchedule( @Param("student") Student student); }
⚡ 고급 쿼리 기법:
• JPQL로 복잡한 시간 충돌 로직
• @Param으로 가독성 향상
• 메소드명 기반 자동 쿼리 생성

🖥️ 실시간 프론트엔드 검증

// 실시간 학점 계산 및 제한 검사 const MAX_CREDITS = 18; // 학점 상한선 const enrollments = []; // 기존 수강 과목 const selectedCourses = []; // 선택한 과목들 function calculateTotalCredits() { let total = 0; // 기존 수강 학점 계산 enrollments.forEach(enrollment => { const credits = parseInt(enrollment.subject.credits) || 0; total += credits; }); // 새로 선택한 과목 학점 계산 selectedCourses.forEach(course => { const credits = parseInt(course.credits) || 0; total += credits; }); return total; } // 실시간 시간 충돌 검사 function hasTimeConflict(newCourse) { const allCourses = [...enrollments, ...selectedCourses]; for (let existing of allCourses) { // 같은 요일인지 확인 if (existing.day !== newCourse.day) continue; // 시간을 분 단위로 변환 const newStart = timeToMinutes(newCourse.startTime); const newEnd = timeToMinutes(newCourse.endTime); const existingStart = timeToMinutes(existing.startTime); const existingEnd = timeToMinutes(existing.endTime); // 시간 겹침 검사 (3가지 경우) if ((newStart < existingEnd && newEnd > existingStart)) { return { conflict: true, conflictWith: existing.subject.name, conflictTime: `${existing.startTime}-${existing.endTime}` }; } } return { conflict: false }; } // 수강신청 버튼 클릭 이벤트 function enrollCourse(courseId, courseName, credits) { // 1. 학점 초과 검사 const newCredits = calculateTotalCredits() + parseInt(credits); if (newCredits > MAX_CREDITS) { showError(`학점 초과: ${newCredits}/${MAX_CREDITS}학점`); return false; } // 2. 시간 충돌 검사 const conflictCheck = hasTimeConflict(course); if (conflictCheck.conflict) { showError(`시간 충돌: ${conflictCheck.conflictWith}`); return false; } // 3. 서버로 요청 전송 fetch('/student/enroll', { method: 'POST', headers: { 'Content-Type': 'application/x-www-form-urlencoded' }, body: `timetableId=${courseId}` }).then(response => { if (response.ok) location.reload(); }); }
⚡ 클라이언트 사이드 실시간 검증:
• 학점 체크: 18학점 상한선 실시간 계산
• 시간 충돌: 분 단위 정밀 계산으로 겹침 검사
• UX 최적화: 서버 요청 전 클라이언트 검증

🚧 주요 도전과제 및 해결방안

🔴 VPC 네트워킹 문제

문제: EKS 클러스터와 RDS가 서로 다른 VPC에 생성되어 통신 불가

에러: vpc-0106946af7a80228d (EKS) ↔ vpc-0a532f02a6fa732c6 (RDS)

✅ 해결: VPC 피어링 연결 + 라우팅 테이블 업데이트로 cross-VPC 통신 구현
🔴 Docker 이미지 불일치

문제: GitHub Actions에서 잘못된 DockerHub 리포지토리 참조

에러: seokheounjo/course-registration → procof/course-registration

✅ 해결: DockerHub 사용자명 불일치 수정 및 시크릿 변수 재설정
🔴 데이터베이스 초기화 충돌

문제: data.sql 중복 키 에러로 애플리케이션 시작 실패

에러: Duplicate entry 'CS101' for key 'subject.UK_subject_code'

✅ 해결: SPRING_SQL_INIT_MODE=never 환경변수로 SQL 초기화 스크립트 비활성화
🔴 EC2 SSH 접근 권한

문제: GitHub Actions에서 EC2 SSH 연결 시 사용자명 오류

에러: ec2-user@ip → ubuntu@ip (Ubuntu AMI 사용)

✅ 해결: AMI별 기본 사용자명 확인 후 ubuntu로 수정
