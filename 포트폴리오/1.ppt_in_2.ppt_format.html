시스템 아키텍처

MVC 아키텍처 패턴

  • Controller 계층: HTTP 요청 처리 및 응답 관리
  • Service 계층: 비즈니스 로직 구현
  • Repository 계층: 데이터 접근 로직
  • Entity 계층: 데이터베이스 엔티티 매핑
  • DTO 계층: 데이터 전송 객체

아키텍처 장점

계층별 책임 분리로 유지보수성과 확장성을 확보하고, Spring Boot의 DI를 통한 느슨한 결합 구현

Controller → Service → Repository → Entity ↓ ↓ ↓ ↓ HTTP 요청 비즈니스 데이터 접근 DB 매핑 처리 및 로직 구현 로직 구현 객체 관리 응답 관리

동시성 제어 및 트랜잭션

트랜잭션 관리

  • @Transactional 애노테이션을 통한 선언적 트랜잭션
  • 서비스 계층에서 트랜잭션 경계 설정
  • 데이터 일관성 보장
  • 롤백 처리를 통한 데이터 무결성 유지

수강신청 프로세스

중복 신청 방지 → 정원 확인 → 수강 등록 → 트랜잭션 커밋

@Transactional public boolean enrollStudent(String studentId, Long subjectId) { // 1. 중복 신청 검사 if (enrollmentRepository.existsByStudentIdAndSubjectId(studentId, subjectId)) { return false; } // 2. 정원 확인 로직 Subject subject = subjectRepository.findById(subjectId); long currentEnrollment = enrollmentRepository.countBySubjectId(subjectId); if (currentEnrollment >= subject.getCapacity()) { return false; } // 3. 수강 등록 Enrollment enrollment = new Enrollment(student, subject); enrollmentRepository.save(enrollment); return true; }
